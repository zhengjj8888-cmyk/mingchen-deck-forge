# 明辰 Deck Forge

把文章、产品资料、数据摘要或一个观点，做成一套**可直接在浏览器演示的 HTML 幻灯片**。

它不是“把一段文字塞进 PPT”。这个 Skill 先帮助你找到观众真正应该记住的一句话，再把内容拆成有节奏的页面、统一的视觉语言和能正常翻页的网页演示稿。

本仓库中的流程、模板和检查脚本由明辰原创设计与维护，并以 [MIT License](LICENSE) 开源。

## 它能帮你完成什么

| 你手上的材料 | 最终得到的东西 |
| --- | --- |
| 一篇长文章或一份会议记录 | 6–10 页的分享型演示稿 |
| 一个产品想法或发布资料 | 有开场、价值、证据和行动页的发布稿 |
| 一份数据摘要 | 结论优先、口径明确的汇报页 |
| 一段课程或演讲大纲 | 可讲、可翻页、可打印的讲义 |
| 一组截图和说明 | 有解释顺序的产品演示页面 |

默认输出是一个自包含的 `.html` 文件：保存后双击即可在浏览器打开，按左右方向键翻页，也可以直接打印成 PDF。它**不是** PowerPoint 的 `.pptx` 生成器。

## 第一次使用：按这 4 步来

### 1. 安装到 Codex 的 Skills 目录

在终端中运行：

```bash
cd ~/Desktop
git clone https://github.com/zhengjj8888-cmyk/mingchen-deck-forge.git
mkdir -p ~/.codex/skills
cp -R mingchen-deck-forge ~/.codex/skills/
```

如果你已经下载了 ZIP，把解压后的 `mingchen-deck-forge` 整个文件夹复制到 `~/.codex/skills/` 也可以。

### 2. 重启 Codex

重启后，Codex 才会识别新安装的 `$mingchen-deck-forge` Skill。

### 3. 直接这样提需求

把内容粘贴进去，并告诉它受众和用途。比如：

```text
使用 $mingchen-deck-forge。

把下面这篇文章做成一份 8 页的中文网页演示稿，
听众是第一次接触这个主题的创业者，时长 8 分钟。
先给我页面表，再生成一个可直接打开的 HTML 文件。
整体要克制、干净，结尾给出下一步行动。

[把文章粘贴在这里]
```

### 4. 打开与检查成品

把生成的内容保存为 `my-deck.html`，双击用浏览器打开。左右方向键翻页；打印时浏览器会按页输出。

提交或分享前，运行随附检查工具：

```bash
python3 ~/.codex/skills/mingchen-deck-forge/scripts/check_deck.py ./my-deck.html
```

看到 `OK: structural checks passed`，表示 HTML 的基本结构已通过检查。它不替代人工检查：数字、来源、标题截断和演讲节奏仍应逐页确认。

## 三个可以直接复制的提示词

### 把文章做成分享稿

```text
使用 $mingchen-deck-forge，把下面文章做成 7 页分享稿。
目标是让团队在 10 分钟内理解：为什么现在要做这件事。
不要照搬文章顺序。每页只讲一个判断，必要处标注来源和日期。
先输出页面表，确认后生成可直接演示的 HTML。
```

### 做产品发布页

```text
使用 $mingchen-deck-forge，为「[产品名]」做一套 9 页产品发布网页演示稿。
受众是潜在客户。主线是“他们现在卡在哪里，我们怎样解决，凭什么可信”。
品牌色是 #FF6B3D，16:9。不要编造客户、数据和案例；
不确定的信息请单独列为待补充项。
```

### 做数据复盘页

```text
使用 $mingchen-deck-forge，把这份周报数据做成 6 页管理层复盘。
第一页直接给结论；后面依次解释变化、证据、风险和下周动作。
所有同比、环比和时间范围必须写清，不能根据缺失数据推测。
最后生成自包含 HTML，并说明检查结果。
```

## 它是怎样工作的

```text
原始材料
   ↓
一句核心判断 + 受众目标
   ↓
页面表（每页承担一个认知动作）
   ↓
统一视觉系统（颜色、字号、间距、图表）
   ↓
可翻页的 HTML 幻灯片
   ↓
结构检查 + 逐页人工校对
```

最关键的一步是页面表。每一页都应写清：

| 字段 | 要回答的问题 |
| --- | --- |
| 页码 | 这是叙事的第几步？ |
| 观众要理解什么 | 看完这一页，认知发生什么变化？ |
| 结论标题 | 用一句结论，而不是“市场分析”式目录标题 |
| 证据/素材 | 依据来自哪里？数据口径和日期是什么？ |
| 版式 | 大字结论、对比、流程、卡片还是图表？ |
| 时长 | 讲这一页预计花多久？ |

## 自带模板：不用等 AI，也能马上开始

仓库内的 [assets/minimal-deck.html](assets/minimal-deck.html) 是一份从零写的三页起步模板：自包含、16:9、支持方向键翻页和打印。

把它复制为自己的文件：

```bash
cp ~/.codex/skills/mingchen-deck-forge/assets/minimal-deck.html ./my-deck.html
python3 ~/.codex/skills/mingchen-deck-forge/scripts/check_deck.py ./my-deck.html
```

打开 `my-deck.html` 后，优先改这几处：

1. 在 `<title>` 中写演示稿名称。
2. 在 CSS 的 `:root` 中替换 `--ink`、`--paper`、`--accent` 三个颜色变量。
3. 用自己的 `<section class="slide">` 替换三页示例内容。
4. 复制一个 `section` 就能增加新页，同时改好右下角页码。
5. 演示时按 ← / →；浏览器打印时会一页一页输出。

## 做出“能讲”的演示稿：几个原则

- 先写“观众最后能复述什么”，再想视觉。
- 首页只给一个值得继续看的判断，不把背景介绍堆在第一屏。
- 一页只承担提出、解释、比较、证明或行动中的一个动作。
- 信息多时拆成两页，采用“先结论，后证据”；不要靠缩小字号硬塞。
- 做比较时统一维度和单位；出现数字时写清来源、日期和口径。
- 没有用户提供的事实、图片或授权素材时，使用排版、色块、CSS 或 SVG 表达；不要补造案例和数据。
- 动效只服务于顺序和强调。模板保持轻量，避免为了炫技让演示失焦。

## 边界与常见问题

**需要会写代码吗？**  
不需要。安装后把材料和目标说清楚即可。模板适合愿意自己微调颜色、文字和页面内容的人。

**能生成 .pptx 吗？**  
本 Skill 的稳定交付物是 HTML。优点是无需额外软件、容易分享、可在浏览器演示和打印；若必须交付 PowerPoint，请在需求里明确提出，另行处理。

**可以自动找网络图片吗？**  
默认不会。为了避免版权、来源和风格问题，优先使用你提供的素材或纯排版图形。需要外部素材时，请先确认授权和来源。

**检查脚本通过就可以发布吗？**  
还不够。脚本只检查 HTML 骨架与页面数量。发布前仍要检查事实、数字、引用、错别字、对比度和实际讲述节奏。
